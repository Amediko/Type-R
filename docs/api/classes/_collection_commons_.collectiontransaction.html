Class CollectionTransaction

In Collections, transactions appears only when add remove or change events might be emitted. reset doesn't require transaction.

Transaction holds information regarding events, and knows how to emit them.

Two major optimization cases. 1) Population of an empty collection 2) Update of the collection (no or little changes) - it's crucial to reject empty transactions.
